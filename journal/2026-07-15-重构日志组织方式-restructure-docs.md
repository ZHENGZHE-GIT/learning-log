# 重构日志组织方式 + 建立任务工作流

> 📅 2026-07-15 | 🏷️ 项目优化 · 日志规范 · Git · CLAUDE.md

## 背景

原本按天合并日志（一个文件塞一天所有任务），导致同一文件内混入不相关内容（协作体系搭建 + 教程蒸馏）。同时每次任务结束时的收尾步骤（写日志、更新 README、commit）靠记忆执行，容易遗漏。需要一个标准化的任务组织和收尾机制。

---

## 完成事项

### 1. 日志组织方式：从"按天"改为"按任务"

**做了什么：**
- 将 Day 2 文件拆分为两个独立任务日志：协作体系搭建 + 尚硅谷教程蒸馏
- 删除所有日志 frontmatter 中的 `day` 编号字段
- 删除日志标题中的 `# Day N：` 前缀
- 更新 `templates/daily-log.md` 模板
- 更新 `CLAUDE.md` 和 `memory/project-learning-log.md` 中的规范描述

**学到了什么：**
- 按任务组织比按天组织更适合学习日志——查找时按主题而非日期回忆
- 文件名保留日期前缀用于排序，但内容完全按任务独立
- 同一天多个任务 = 多个文件，互不干扰

### 2. README 标签索引去 Day N 化

**做了什么：**
- 标签索引中所有 `[Day 1]`/`[Day 2]` 替换为任务名简称（`环境搭建`/`协作体系`/`知识蒸馏`）
- 新增"参考文档"固定区域，放置速查手册和教程原文链接

**学到了什么：**
- 标签链接文字应该描述"内容是什么"而非"第几天做的"——后者对查找没有帮助

### 3. 建立任务完成工作流

**做了什么：**
- 在 `CLAUDE.md` 新增"任务完成工作流"章节，定义 AI 在任务结束时的 4 个强制步骤：
  1. 生成任务日志
  2. 更新 README.md
  3. 提交到本地仓库
  4. 展示任务摘要
- 包含回退机制说明（`git revert`/`git reset`）
- 创建 `.claude/commands/task-done.md` 自定义命令

**学到了什么：**
- 把工作流写进 CLAUDE.md 比靠记忆可靠——AI 每次新会话自动加载
- 每个任务独立 commit 的好处：`git log` 即任务历史，出错时精确回退

### 4. 新增常用指令速查手册

**做了什么：**
- 创建 `docs/常用指令速查.md`，按 7 大分类索引：
  - Claude Code 会话命令、模型与费用、Git 任务工作流、自定义命令、键盘快捷键、Shell 常用命令、文件引用技巧
- README 新增"参考文档"区域引用该手册
- CLAUDE.md 外部依赖表中添加该文档路径

**学到了什么：**
- 新手不需要记住所有命令——有一个按分类索引的速查页面就够了
- 速查手册应该是"活文档"，新命令随时往里加

## 踩坑记录

| 问题 | 原因 | 解决 |
|------|------|------|
| 蒸馏内容拆分后原文件残留大量文本 | Edit 工具只能匹配有限长度，蒸馏内容太长需要多次删除 | 用 `head -n 161` 直接截断文件（确认行号后） |
| `尚硅谷AI Coding教程.md` 被意外删除 | 原因不明，`git add -A` 时检测到文件消失 | `git checkout HEAD --` 从历史恢复 |
| 标签索引中还有一个 Vibe Coding 写成了 Day 1（实际应为 Day 2） | 手动编辑时的笔误 | `replace_all` 全量替换整个标签块 |
