# 每月合成

请按以下步骤完成本月的知识合成工作：

## 步骤

1. **扫描本月所有 journal**
   - 列出本月（过去 30 天）所有 `journal/` 日志
   - 统计：日志数量、涉及标签、总计踩坑数

2. **识别跨任务模式**
   - 对比多篇日志的"踩坑记录"：是否有重复出现的问题？
   - 对比"完成事项"：是否有不同任务中学到的互补知识？
   - 对比"下一步计划"：是否有跨任务的趋势或方向？

3. **生成月度总结**
   - 在 `journal/` 创建文件：`YYYY-MM-DD-月度学习总结-monthly-review.md`
   - 内容包含：
     - **本月概览**：日志数、关键标签、学习时长估计
     - **核心收获**：本月最重要的 3-5 个认知提升
     - **重复踩坑**：如果同一个问题出现了两次以上，提炼为 notes
     - **趋势观察**：学习重心在往哪个方向移动？
     - **下月计划**：基于"下一步计划"汇总的未来方向

4. **更新 _status.md**
   - 将本月已完成的条目移到"已完成"归档
   - 根据月度总结更新"下一步"

5. **可选：提交**
   - `git add -A && git commit -m "docs: 月度合成 $(date '+%Y-%m')"`
   - 不 push

> 提示：月度合成不是"把每周回顾再写一遍"，而是**站高一层看趋势**——这个月我整体在往哪个方向走？哪些坑我反复踩？
