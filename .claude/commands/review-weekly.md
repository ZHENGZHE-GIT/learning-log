# 每周回顾

请按以下步骤完成本周的知识蒸馏工作：

## 步骤

1. **清空 inbox**
   - 读取 `_inbox.md` 的"待处理"列表
   - 逐条判断归属：
     - 可执行的任务想法 → 创建 `journal/` 日志或在已有日志的"下一步计划"中添加
     - 概念性理解/学到的东西 → 写入 `notes/` 对应子目录（按 Agent 学习阶段选择）
     - 外部链接/工具 → 补充到 `references/` 对应文件
   - 处理完毕后清空"待处理"列表

2. **蒸馏本周 journal**
   - 扫描本周（过去 7 天）的 `journal/` 日志
   - 从每篇日志的"完成事项"中提取 **学到了什么**
   - 从"踩坑记录"中提取关键教训
   - 将提取的知识点写入 `notes/` 对应子目录：
     - 每个知识点一个独立段落
     - 标注来源日志（如 `来源: journal/2026-07-16-xxx.md`）
     - 如果已有相关笔记，追加到已有文件而非新建

3. **更新 _status.md**
   - 将本周完成的"下一步"条目勾掉
   - 从本周日志的"下一步计划"汇总新条目
   - 更新"待回顾"区域

4. **可选：提交**
   - 如果产生了新文件或修改，`git add -A && git commit -m "docs: 每周回顾 $(date '+%Y-%m-%d')"`
   - 不 push

> 提示：蒸馏的核心不是复制粘贴，是**用自己的话重新表达**。从"我做了什么"提炼出"我学到了什么"。
