## 一、Claude Code 会话命令

### 核心高频

| 命令 | 作用 | 何时用 |
|------|------|--------|
| `/plan` | 进入只读规划模式 | 复杂任务起手，先出方案再动手 |
| `/compact` | 压缩上下文为摘要 | 对话变长、AI 变慢/变笨时 |
| `/clear` | 彻底清空对话 | 一个任务结束，开始新任务 |
| `/context` | 查看上下文占比 | 想看看是什么在吃掉 token |
| `/rewind` | 回滚 cc 的修改 | 代码被改坏了，选回滚方式 |
| `/init` | 自动生成 CLAUDE.md | 新项目第一件事 |

### 审查与检查

| 命令 | 作用 |
|------|------|
| `/review` | AI 代码审查（每个功能完成后用） |
| `/security-review` | 安全检查（认证模块 + 上线前用） |
| `/verify` | 验证代码正确性 |

### 记忆与配置

| 命令 | 作用 |
|------|------|
| `/memory` | 管理记忆 / 开启 Auto Memory |
| `/status` | 查看当前会话状态（模型、token 消耗） |

### 对话技巧

| 命令 | 作用 |
|------|------|
| `/btw` | "顺便问一句"，不污染主上下文 |
| `/resume` | 恢复之前的会话 |
| `/simplify` | 派 3 个子 Agent 从代码质量/性能/复用性三个角度优化代码 |

## 二、模型与费用

### 切换模型

| 命令 | 作用 |
|------|------|
| `/model` | 交互式选择模型 |
| `/model sonnet` | 切到 Sonnet（日常主力） |
| `/model opus` | 切到 Opus（复杂推理） |
| `/model haiku` | 切到 Haiku（轻量快速） |

### 费用监控

| 命令 | 作用 |
|------|------|
| `/cost` | 查看当前会话花费 |

### 用哪个模型

```
简单任务（补全、格式化）  → Haiku
日常开发（功能实现、Bug） → Sonnet（默认）
复杂任务（架构、算法）    → Opus
```

## 三、Git 任务工作流

### 开始任务前

```bash
git stash          # 保存当前草稿
```

### 完成任务后

```bash
git add -A
git commit -m "docs: 任务标题"
```

### 查看任务历史

```bash
git log --oneline          # 每个任务一个 commit，一目了然
```

### 回退

```bash
git revert <commit>         # 安全回退：撤销某个 commit，保留历史
git reset --hard <commit>   # 硬回退：回到某个 commit，丢弃后续所有改动
git checkout .              # 放弃当前未提交的修改
```

> **记忆口诀**：任务前 `stash`，任务后 `commit`。改坏了 `revert`，不想要了 `reset`。

## 五、键盘快捷键

| 快捷键 | 作用 |
|--------|------|
| `Enter` | 发送消息 |
| `Ctrl + Enter`（Windows） | 换行（不发送） |
| `Ctrl + C` | 中断当前操作 |
| `Esc` | 取消正在生成的内容 |
| `Esc × 2`（双击） | 启动 `/rewind` 回滚界面 |
| `Shift + Tab` | 切换权限模式（Normal → Auto-Accept → Plan） |
| `Ctrl + B` | 后台运行当前命令 |
| `Ctrl + O` | 查看 Auto Memory 记录 |

### 三种权限模式

| 模式 | 行为 | 切入方式 |
|------|------|---------|
| Normal | 每次操作都确认 | 默认 |
| Auto-Accept | 自动执行 | `Shift+Tab` ×1 |
| Plan Mode | 只读，只能分析 | `Shift+Tab` ×2 或 `/plan` |

## 七、文件引用技巧

| 写法 | 作用 |
|------|------|
| `@src/auth/login.ts` | 引用特定文件给 AI 当上下文 |
| `@docs/feature-spec.md` | 引用需求文档让 AI 按文档实现 |
| 拖拽文件到对话框 | 多模态：截图、设计稿 |
| `Ctrl+V` 粘贴图片 | 贴报错截图让 AI 诊断 |

> **核心原则**：指令越具体 + 精准 @ 文件，AI 消耗的 token 反而越少，效果越好。

## 附录：Claude Code 启动参数

```bash
claude                              # 默认启动
claude --model sonnet               # 指定模型启动
claude -c                           # 继续上次会话
claude --permission-mode plan       # 启动即进入 Plan Mode
claude -p "单次提问"                # 单次执行模式（不进入交互）
claude --dangerously-skip-permissions  # 危险模式，全程不确认
```
